Проект №2 - PyGame. Название проекта - "Hold'em'All"

Постановка задачи:
	- Как большой фанат жанра TowerDefence, я решил написать небольшой прототип игры в этом жанре. Задача: реализовать
	все основные механики, присущие играм подобного жанра, и подготовить почву для дальнейшего развития

Запуск приложения:
	Из сторонних библиотек требуется только pygame. Модули sys и math, модули enemies, towers и projectiles самодельные.
	Запускать файл main.py.

Как играть:
	- После запуска игры появляется стартовое окно с кратким описанием. При нажатии на кнопку "Начать" начинается игра
	- перед игроком появляется клетчатое поле. Бежевые клетки - это маршрут для врагов, зеленые - фоновое окружение, 
		серые - места для строительства башен, ярко-синяя - это "портал", куда нельзя пускать врагов
	- Изначально игроку дано 75 золота. Первая волна стартует через 10 секунд после запуска игры.
	- Чтобы построить башню, нужно кликнуть по серому полю. На строительство требуется 50 золота.
	- Подсказка: для прохождения второго уровня потребуется больше одной башни, хотя для демо-режима их радиус стрельбы оставлен большим.
	- Подсказка 2: я оставил пасхалку: по щелчку по черной области в игровом окне одна из башек выполняет внеочередной выстрел. То есть можно расстрелять врагов "вручную".
	- Если 10 врагов достигнут портала, появится окно GAME OVER
	- Если вы выстоите 3 волны, сохранив хотя бы одну жизнь, начнется следующий по порядку уровень
	- Если вы выстоите последний уровень (сейчас их два), игра завершится и вернется на стартовое окно.

Реализованные механики и принципы игры:
	- вся информация об уровне (размер, карта, количество волн и так далее) считывается из текстовых файлов
		Алгоритм считывания универсальный, и для дальнейшего добавления уровней достаточно лишь добавлять файлы с ними, 
		а в программе поменять нужно лишь само количество уровней.
	- Волны появляются через заданный интервал. Враги внутри волны - тоже.
	- Башни стреляют каждая в своем темпе (скорострельность, урон и радиус указаны в атрибутах башен)
	- У врагов задана скорость, здоровье
	- Классы Tower и Enemy написаны в общем виде, и от них можно отнаследовать более сложные и интересные типы башен
		и противников, придумывать новые механики
	- Игрок теряет одну из жизней, когда враг попадает в портал
	- Башни выбирают цель по принципу "ближайший к выходу", но с учетом ограниченности своего радиуса стрельбы
	- Снаряд имеет определенную скорость и урон, который получает от породившей его башни. При столкновении с врагом наносит урон
	- Второй уровень чуть побольше, в нем на одну волну больше, запускается он сразу по завершении первого.
	


Выводы:

		-условие поражения
		-переход на следующий уровень
		-деньги
		-строительство башен
		-спаун волн врагов
		-стрельба
		-уход врагов в портал
